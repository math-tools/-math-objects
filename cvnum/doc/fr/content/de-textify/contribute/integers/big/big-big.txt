===============================
Nommer des ¨nbres "très grands"
===============================

La méthode retenue pour nommer des ¨nbres "très grands" est basique.
Prenons le cas de l'espagnol qui permet de parler de "millones de millones de millones"
((
    Nous ne répondrons pas à la question épineuse de l'utilité de ceci.
)).
Cette régle est indiquée comme suit via la clé optionnelle cvnum::``big`` du bloc cvnum::``general``.

cvnum::
    ---
    file    = ES/ES/integers.txt
    extract = yes
    ---

    general::
    // ... etc.
        big = mill... -> ... de millones

    group::
    // ... etc.
        6 = "un :if: d = 1 :else: [d]"
            mill"ones :if: d > 1 :else: ón"
            [r]


Le mécanisme disponible est basique.

    1) cvnum::``big = mill...`` dans cvnum::``big = mill... -> ... de millones`` indique de capturer tout mot commençant par "mill" dans le nom du nombre. Ici les trois ¨pts de suspension indiquent toute séquence de caractères sans espace.

    1) cvnum::``... de millones`` dans cvnum::``big = mill... -> ... de millones`` demande de compléter les "millones" et "millón" repérés par autant de "de millones" que nécessaire.
    Ici les trois ¨pts de suspension indiquent toute occurence de "millones" ou "millón" (car ce sont les seuls mots produits par les règles de nommage qui commencent par "mill").


note::
    Si la clé cvnum::``big`` n'est pas utilisée alors la ¨trad sera refusée pour les ¨nbres trop grands.
